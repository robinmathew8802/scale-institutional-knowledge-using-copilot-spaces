# Persona Assignment & Onboarding Checklist

Purpose: ensure that for each project we explicitly assign core and optional personas, document responsibilities, and remove ambiguity.

- [ ] Project Manager assigned
- [ ] Product Manager assigned
- [ ] Developers (owner/lead) assigned
- [ ] QA Lead assigned (or QA responsibilities assigned)
- [ ] Release Manager assigned (or release responsibilities assigned)
- [ ] UX / Design Lead assigned (if UX work is required)
- [ ] Stakeholder Advocate assigned (if external stakeholders involved)
- [ ] Platform / SRE Lead assigned (if production/platform impact)
- [ ] Persona owners added to project README and kickoff notes
- [ ] Contact handles and backup contacts listed
- [ ] Acceptance criteria for role handoffs documented (e.g., QA sign-off required before release)
